# Evidence

**Purpose:**  
Store supporting documentation for work search activities (application confirmations, emails, interview invites, etc.).

**Use this when:**
- Saving proof of job applications
- Storing email confirmations
- Keeping interview invitations
- Preparing for potential TWC audits

---

## File Naming Convention

- Format: `YYYY-MM-DD_CompanyName_Description.ext`
- Examples:
  - `2026-01-18_HPE_Application.pdf`
  - `2026-01-20_LinkedIn_Message.png`
  - `2026-01-22_Interview_Invite.pdf`

---

## Organization

- **Root level:** Application confirmations, emails, screenshots
- **Interviews/:** Interview-related documents (invites, confirmations, notes)
- **Applications/:** Application confirmations and receipts

---

## Best Practices

1. **Save immediately:** Don't wait to save evidence
2. **Be descriptive:** Use clear file names that identify the activity
3. **Reference in log:** Link evidence files in `Work_Search_Log.md` notes column
4. **Keep organized:** Use subfolders for different types of evidence

---

## Integration

- **Referenced by:** `Work_Search_Log.md` (in notes column)
- **Referenced by:** `Weekly_Certifications/` files
